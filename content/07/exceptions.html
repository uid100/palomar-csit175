Exceptions and Exception Handling

An exception in programming refers to an event that disrupts the normal flow of a program's instructions during its execution. It typically occurs when an error or unexpected condition arises that prevents the program from continuing its operation as expected.

Exception handling mechanisms allow programmers to anticipate and gracefully deal with such errors, providing alternative paths for the program to follow when exceptions occur. This helps in maintaining program stability and robustness, as well as improving user experience by providing informative error messages.

Controlled Execution

In this study of Python, we began by considering sequential instructions, giving the computer step-by-step instructions to complete simple tasks. We expanded the discussion add conditional branching and iteration using for and while loops before discussing modularization and moving blocks of code to functions to make our code more robust, readable, and reusable.

You have seen how exceptions can occur, and if left unhandled will interrupt program execution.

In this chapter we will investigate ways to enable the program to recognize when an exception occurs, and to continue to run or recover from an error instead of just halting execution.

Try-Except-Finally

In Python, the try statement is used to enclose the code that might raise an exception. The except block is used to handle the exception, providing a fallback mechanism to deal with errors. The finally block is optional and is always executed whether an exception occurs or not. It's typically used for cleanup actions.

Example:


try:
    # Code that might raise an exception
    result = 10 / 0
except ZeroDivisionError:
    # Handling a specific exception
    print("Division by zero error!")
finally:
    # Cleanup code
    print("Execution completed.")

Handling Single Exception

You can handle a specific exception type using a single except block. This block will only catch the specified exception.

Example:


try:
    # Code that might raise an exception
    file = open("nonexistent_file.txt", "r")
except FileNotFoundError:
    # Handling a specific exception
    print("File not found error!")

Handling Multiple Exceptions

You can handle multiple exception types using multiple except blocks or a single except block with multiple exception types listed.

Example using multiple except blocks:


try:
    # Code that might raise an exception
    result = 10 / 0
except ZeroDivisionError:
    # Handling a specific exception
    print("Division by zero error!")
except Exception as e:
    # Handling other exceptions
    print(f"An error occurred: {e}")

Example using a single except block with multiple exception types:


try:
    # Code that might raise an exception
    file = open("nonexistent_file.txt", "r")
except (FileNotFoundError, PermissionError):
    # Handling multiple exceptions
    print("File error occurred!")
